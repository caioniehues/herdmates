Task close per standing rule. Commit 6c4cfaa local main, #98 closed,

## Infra discoveries (live-verified, filed as #101)

- `plugin.pane.open` never spawns relative manifest command paths
  (`No viable candidates found in PATH`) — for EVERY entrypoint, proven
  by control test on pre-existing `focus` and an absolute-path spike
  (only variable changed; spawn succeeded immediately, pane w1A:p1F).
  Explains why focus's live E2E historically never ran.
- Herdr's manifest cache is stale until explicit `herdr plugin link`;
  `plugin list` and pane-open resolution never live-read the TOML.

## Coordination learnings (native-team substrate)

- **Mid-turn SendMessage dies at idle** (3x this task): a message
  delivered while a teammate is mid-turn queues but the teammate can
  idle without draining it; a send to an already-idle teammate reliably
  resumes it. Rule: idle notification after a mid-turn send = assume
  unprocessed; verify at the teammate's plan-file tail; re-wake pointing
  at the queued msg id.
- **DONE contract requires an inbox-drained section** listing each lead
  decision received + how applied. builder-98's first DONE re-asked a
  question already answered in its inbox; the drain requirement caught
  every subsequent crossing.
- Control-test method for "is this our bug": run the identical operation
  against a pre-existing merged artifact (focus pane) before blaming the
  new code.

## Design learnings

- Task-file mtime is since-last-write, not task age — honesty doctrine
  forced "updated Xs ago" labeling, a deliberate wording deviation from
  #98's literal "elapsed".
- Manifest actions have static argv → any pane entrypoint needs ambient
  team resolution; `resolve_team` (explicit wins / exactly-one fallback /
  ambiguous hard error with candidates) is the honest minimal shape.
  Carried: recorder adoption.
