# Wave 7 learnings — state consistency + docs pair (2026-07-15)

Retrospective for #28+#29 (PR #36) and #16+#34 (PRs #35, #37). Per standing
rule: formalize learnings at every wave close.

## Headline discovery: codex workers ARE skill-drivable — via `$`, not `/`

Caio's hunch, live-verified on codex-cli 0.144.4: typing `$` in the codex
composer opens a skill/template picker listing the installed skill set (Ask
Matt, Code Review, Diagnosing Bugs, …), and `$ask-matt <question>` loads the
skill and answers from its content. Claude-style `/name` fails
(`Unrecognized command`) — `/` and `$` are separate surfaces. Consequences:
- Skill-dependent tasks can route to codex workers directly (`$<skill-name>`
  in the brief); inlining skill content is now optional hardening.
- First doc shipped wrong (#35 said codex can't use skills) because the
  probe used `/` only. **Rule: probing one invocation surface proves nothing
  about the others — enumerate the composer's prefix triggers (`/`, `$`, `@`)
  before writing a capability claim.**
- Reference: `skills/codex-prompting/SKILL.md` (corrected in PR #37).

## Process learnings

### Verify a worker's ground-truth claims with the tool that CAN'T submit
The codex slash-list in #35 looked wrong (claude-like names). Probing with
`pane run "/"` failed — it submits, triggering the first menu action.
`herdr agent send` (writes WITHOUT submitting) rendered the raw popup and
proved the list correct. First legitimate use of `agent send`; ADR-0008's
"never a messaging channel" rule stands — this is a UI-probe, not messaging.

### Stopped-not-done nudge works as designed
K went `done` with no report mid-wave (turn ended after a progress ping).
One `msg` restating the completion contract resumed him to a full PR. The
monitor pattern (report-file existence + terminal-state check) caught it
within 30s.

### Review depth scaled right
Docs PRs (#35/#37): inline coordinator review + ground-truth spot-probes.
State-consistency PR (#36): full adversarial pane review — found a real
MEDIUM (recovery double-submit window) plus a TOCTOU that unit tests
couldn't see. Same lesson as wave 6: reviewers must attack crash windows,
not read for style.

### Accepted-residual reasoning belongs in the review record
PR #36's double-submit window is narrowed (RPC-return → flock'd write), not
closed — unclosable without an atomic submit+persist herdr doesn't offer;
checkpoint-before-submit would invert into permanently-unbriefed (worse).
Recording WHY the residual is accepted prevents a future "bug" report.

## Mechanics learnings

- `adopt --team` refuses while another run is active — kill the finished
  wave's run before ad-hoc probe teams (`pass --run or kill it`).
- Manufactured crash states are honest probes: hand-editing run.toml to
  `lifecycle=pending` + `checkpoint=brief_submitted` reproduced the #29
  crash window exactly; recovery flipped it to running with zero
  resubmission, and re-run reported healthy/no-op.
- GitHub auto-deletes merged PR head branches (repo setting) — batch
  `push --delete` may partially fail; delete per-branch and verify with
  `git ls-remote --heads`.

## Verified product wins

- No production path writes run.toml without the flock (compile-time
  `#[cfg(test)]` visibility + guard test); mid-adopt hook writes survive
  adopt/msg/kill saves both directions.
- Stuck-pending adoptee recovery: live-verified recover → running without
  resubmit; idempotent second run says "already healthy; nothing to do".
- Kill's end-of-run decision now made inside the locked closure (TOCTOU
  closed).
- Relink procedure + codex prompting reference shipped as durable docs.
